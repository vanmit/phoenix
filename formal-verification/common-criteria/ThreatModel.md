### 3.2 威胁源详细描述

- **动机**: 经济利益（勒索、数据出售）、破坏声誉、间谍活动
- **能力**: 熟练掌握漏洞利用、社会工程学、自动化工具
- **攻击面**: 网络接口、文件解析、第三方依赖

#### TS002: 恶意内部人员
- **动机**: 经济利益、报复、竞争
- **能力**: 合法访问权限、了解内部系统
- **攻击面**: 所有系统组件
- **缓解**: 最小权限、审计日志、职责分离

#### TS003: 意外用户错误
- **动机**: 无恶意
- **能力**: 有限
- **攻击面**: 配置错误、误操作
- **缓解**: 安全默认配置、确认对话框、培训

### 4.1 威胁场景矩阵

### 4.2 详细威胁场景

#### T001: 内存敏感数据窃取
- **描述**: 攻击者通过内存转储获取加密密钥或用户数据
- **攻击路径**:
  1. 获取系统访问权限
  2. 使用工具（如 LiME）转储内存
  3. 搜索敏感数据模式
  4. 提取密钥或数据
- **现有控制**:
  - zeroize 库清除敏感数据
  - 内存加密（硬件支持）
  - 最小化敏感数据驻留时间
- **残余风险**: 🟡 中等

#### T002: 文件解析漏洞利用
- **描述**: 攻击者通过恶意资源文件触发缓冲区溢出或代码执行
- **攻击路径**:
  1. 创建恶意资源文件
  2. 诱使用户加载文件
  3. 触发解析漏洞
  4. 执行任意代码
- **现有控制**:
  - Rust 内存安全
  - 输入验证和边界检查
  - 沙箱执行
  - 形式化验证解析器
- **残余风险**: 🟢 低

#### T003: 网络通信窃听
- **描述**: 攻击者截获网络通信获取敏感信息
- **攻击路径**:
  1. 接入同一网络
  2. ARP 欺骗或 DNS 劫持
  3. 截获通信
  4. 解密或分析数据
- **现有控制**:
  - TLS 1.3 加密
  - 证书验证
  - 完美前向保密
- **残余风险**: 🟢 低

#### T008: 供应链攻击
- **描述**: 攻击者通过污染的第三方依赖渗透系统
- **攻击路径**:
  1. 入侵依赖库维护者
  2. 注入恶意代码
  3. 发布更新
  4. 目标系统自动更新
- **现有控制**:
  - 固定依赖版本
  - cargo-audit 扫描
  - 依赖来源验证
  - 构建 reproducible
- **残余风险**: 🟡 中等

## 5. 脆弱性分析 (Vulnerability Analysis)

### 5.1 脆弱性识别

| 脆弱性 ID | CWE | 描述 | 位置 | 严重性 | 状态 |
|-----------|-----|------|------|--------|------|
| V001 | CWE-119 | 缓冲区溢出 | FFI 边界 | 高 | ✅ 已修复 |
| V002 | CWE-416 | 释放后使用 | 内存管理 | 高 | ✅ 已修复 |
| V003 | CWE-362 | 竞态条件 | 并发访问 | 中 | ✅ 已缓解 |
| V004 | CWE-200 | 信息泄露 | 错误消息 | 中 | ✅ 已修复 |
| V005 | CWE-327 | 弱加密 | 测试代码 | 低 | ✅ 已标记 |
| V006 | CWE-770 | 无限制分配 | 资源加载 | 中 | ✅ 已限制 |

### 5.2 脆弱性详细说明

#### V001: FFI 边界检查不足 (已修复)
- **原始问题**: crypto_decrypt 函数未充分验证输入长度
- **修复措施**:
  - 添加 nonce_len 边界检查
  - 添加整数溢出检查
  - 添加密文最小长度检查
  - 形式化验证 (Coq)
- **验证**: Coq 证明 no_integer_overflow 定理

#### V002: 悬垂指针风险 (已修复)
- **原始问题**: audit_entry_get_timestamp 返回内部引用
- **修复措施**:
  - 返回 CString 副本
  - 明确生命周期文档
  - 考虑 arena 分配器
- **验证**: 代码审查 + 内存安全测试

#### V003: 并发竞态条件 (已缓解)
- **原始问题**: Mutex 锁竞争可能导致死锁
- **缓解措施**:
  - TLA+ 形式化验证无死锁
  - 固定锁获取顺序
  - 超时机制
- **验证**: TLA+ 模型检查

## 8. 残余风险 (Residual Risk)

### 8.1 不可完全消除的风险

| 风险 ID | 残余风险描述 | 原因 | 管理措施 |
|---------|--------------|------|----------|
| R001 | 物理内存攻击 | 需要硬件级保护 | 依赖平台安全特性 |
| R004 | 高级供应链攻击 | 无法完全验证所有代码 | 持续监控 + 快速响应 |
| R005 | 人为错误 | 无法完全消除 | 培训 + 检查清单 |

### 8.2 风险接受理由

**R001 (物理内存攻击)**:
- 攻击成本极高（需要物理访问 + 专业设备）
- 目标场景不涉及此类威胁
- 缓解：使用硬件加密内存（如 Intel TME）

**R004 (供应链攻击)**:
- 行业共性问题，无法完全消除
- 已实施最佳实践（固定版本、审计）
- 缓解：持续监控 + SBOM

**R005 (人为错误)**:
- 所有系统固有风险
- 已通过安全默认配置最小化
- 缓解：培训 + 自动化检查

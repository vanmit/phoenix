## 1. 安全策略概述 (Security Policy Overview)

### 1.1 安全目标 (Security Objectives)

Phoenix Engine 的安全策略旨在实现以下目标：

| 目标 ID | 安全目标 | 优先级 |
|---------|----------|--------|
| O.PROTECT_ASSETS | 保护引擎资产（代码、资源、数据）免受未授权访问 | 高 |
| O.AUTHENTICATE | 确保所有操作经过身份验证 | 高 |
| O.AUTHORIZE | 确保所有访问经过授权 | 高 |
| O.AUDIT | 记录所有安全相关事件 | 高 |
| O.INTEGRITY | 保证数据和代码完整性 | 高 |
| O.CONFIDENTIALITY | 保证敏感数据机密性 | 中 |
| O.AVAILABILITY | 保证服务可用性 | 中 |

### 1.2 安全原则 (Security Principles)

1. **最小权限原则 (Least Privilege)**
   - 每个组件仅拥有完成其功能所需的最小权限
   - 权限分级：0 (无权限) ~ 5 (完全权限)
   - 默认权限级别：1 (受限)

2. **纵深防御 (Defense in Depth)**
   - 多层安全控制
   - 边界检查 + 类型安全 + 运行时验证
   - 形式化验证关键路径

3. **零信任架构 (Zero Trust)**
   - 永不信任，始终验证
   - 所有内部和外部请求同等对待
   - 持续验证和监控

4. **安全默认配置 (Secure by Default)**
   - 默认启用所有安全功能
   - 需要显式配置才能禁用
   - 生产环境禁止降低安全级别

## 2. 安全功能规范 (Security Functional Requirements)

### 2.1 身份验证 (FAU_GEN - 审计数据生成)

| 功能 ID | 功能描述 | 实现方式 |
|---------|----------|----------|
| FAU_GEN.1 | 生成审计记录 | AuditLogger 模块 |
| FAU_GEN.2 | 审计记录内容 | 时间戳、级别、组件、线程 ID、消息 |
| FAU_SAR.1 | 审计记录审查 | get_recent_entries() |
| FAU_STG.1 | 审计记录保护 | 只读文件权限、备用日志 |

**审计事件类型**:
- 身份验证事件（登录、登出、失败尝试）
- 授权事件（权限变更、访问拒绝）
- 资源访问（文件、网络、内存）
- 安全违规（边界检查失败、验证失败）
- 系统事件（启动、关闭、错误）

### 2.2 访问控制 (FDP_ACC - 访问控制策略)

| 功能 ID | 功能描述 | 实现方式 |
|---------|----------|----------|
| FDP_ACC.1 | 子集访问控制 | ZeroTrustPolicy |
| FDP_ACF.1 | 基于属性的访问控制 | 数据分类 + 用户许可 |
| FDP_IFC.1 | 子集信息流控制 | 加密通道 |
| FDP_IFF.1 | 简单安全属性 | 安全级别标记 |

**访问控制矩阵**:

| 操作/角色 | Public | Internal | Confidential | Secret |
|-----------|--------|----------|--------------|--------|
| 读取公开数据 | ✓ | ✓ | ✓ | ✓ |
| 读取内部数据 | ✗ | ✓ | ✓ | ✓ |
| 读取机密数据 | ✗ | ✗ | ✓ | ✓ |
| 读取绝密数据 | ✗ | ✗ | ✗ | ✓ |
| 修改数据 | ✗ | 受限 | 受限 | 受限 |
| 删除数据 | ✗ | ✗ | 受限 | 受限 |
| 管理权限 | ✗ | ✗ | ✗ | ✓ |

### 2.3 密码学支持 (FCS_CKM - 密钥管理)

| 功能 ID | 功能描述 | 实现方式 |
|---------|----------|----------|
| FCS_CKM.1 | 密钥生成 | rand::thread_rng() + AES-256 |
| FCS_CKM.4 | 密钥销毁 | zeroize::Zeroize |
| FCS_COP.1 | 加密操作 | aes-gcm (AES-256-GCM) |
| FCS_COP.1 | 完整性操作 | hmac (HMAC-SHA256) |

**密码算法规范**:

```
加密算法：AES-256-GCM
- 密钥长度：256 位
- Nonce 长度：96 位 (12 字节)
- 标签长度：128 位 (16 字节)
- 模式：AEAD (认证加密)

完整性算法：HMAC-SHA256
- 哈希函数：SHA-256
- 密钥长度：256 位
- 输出长度：256 位 (32 字节)

随机数生成：
- 来源：操作系统 CSPRNG
- 库：rand 0.8 + getrandom
```

### 2.4 保护机制 (FPT_RVM - 安全机制)

| 功能 ID | 功能描述 | 实现方式 |
|---------|----------|----------|
| FPT_RVM.1 | 安全机制防 bypass | Rust 类型系统 + 边界检查 |
| FPT_TST.1 | TSF 自测试 | 启动时完整性检查 |
| FPT_TDC.1 | 内部 TSF 数据一致性 | Mutex + Arc |

**安全机制**:

1. **内存安全**
   - Rust 所有权系统防止悬垂指针
   - 边界检查防止缓冲区溢出
   - 零初始化防止信息泄露

2. **线程安全**
   - Mutex 保护共享状态
   - Arc 安全共享
   - 无数据竞争保证

3. **形式化验证**
   - Coq 证明内存分配器安全性
   - Coq 证明加密解密可逆性
   - Isabelle 证明场景图不变量
   - TLA+ 证明无死锁

## 4. 威胁模型 (Threat Model)

### 4.1 威胁源 (Threat Sources)

| 威胁源 | 描述 | 可能性 | 影响 |
|--------|------|--------|------|
| T.EXTERNAL | 外部攻击者 | 中 | 高 |
| T.INTERNAL | 内部恶意用户 | 低 | 高 |
| T.ACCIDENTAL | 意外错误 | 中 | 中 |
| T.ENVIRONMENT | 环境故障 | 低 | 中 |

### 4.2 威胁场景 (Threat Scenarios)

| 场景 ID | 威胁描述 | 缓解措施 |
|---------|----------|----------|
| T.BUFFER_OVERFLOW | 缓冲区溢出攻击 | Rust 内存安全 + 边界检查 |
| T.CODE_INJECTION | 代码注入 | W^X + 沙箱 + 输入验证 |
| T.PRIVILEGE_ESCALATION | 权限提升 | 最小权限 + 零信任 |
| T.DATA_LEAK | 数据泄露 | 加密 + 访问控制 |
| T.DENIAL_OF_SERVICE | 拒绝服务 | 资源限制 + 速率限制 |
| T.TAMPERING | 篡改 | HMAC + 数字签名 |
| T.REPLAY | 重放攻击 | Nonce + 时间戳 |

### 4.3 攻击树 (Attack Tree)

```
攻击目标：获取引擎控制权
├── 利用缓冲区溢出 (已缓解：Rust 内存安全)
├── 利用 UAF (已缓解：所有权系统)
├── 利用竞态条件 (已缓解：Mutex + TLA+ 验证)
├── 利用弱加密 (已缓解：AES-256-GCM)
├── 社会工程学 (需用户培训)
└── 供应链攻击 (已缓解：依赖审计 + 固定版本)
```
